# Принципи програмування

У цьому проєкті продемонстровано дотримання таких принципів:

### 1. KISS (Keep It Simple, Stupid)
Код максимально простий і виконує лише конкретне завдання з приведення типів без створення зайвих абстракцій.
* **Приклад у коді:** [main.go, рядки 20-22](https://github.com/Dishkivskyi/software-construction-lab1/blob/main/ex7.go#L20-L22) — обчислення суми та добутку зроблено прямолінійно.

### 2. YAGNI (You Ain't Gonna Need It)
У програмі немає функціоналу, який не використовується (наприклад, нереалізованих функцій або зайвих імпортів).
* **Приклад у коді:** [main.go](https://github.com/Dishkivskyi/software-construction-lab1/blob/main/ex7.go) — програма містить лише необхідний мінімум для виконання завдання.

### 3. Належне використання статичної типізації (Go Principles)
Код чітко демонструє розуміння того, як Go працює з типами, уникаючи помилок компіляції через неявне приведення.
* **Приклад у коді:** [main.go, рядок 20](https://github.com/Dishkivskyi/software-construction-lab1/blob/main/ex7.go#L20).
